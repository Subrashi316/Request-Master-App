About Project

Request master is a web based application for testing APIs or (Application Programming Interface) online. It is web based application so user can use this application without any installation. It shows each request in an easy to read manner and also shows the respective output from the API. The user can send extra information with the request such as header information and body data with the request.

Objective of project

The main objective of this project is for the user to be able to test his or her API without creating a frontend for it, For example letâ€™s say, a team of engineers have to create a full stack web application and the work is divided among the frontend engineer and the backend engineer. As for a backend engineer, the main work for him is mainly to create the backend. In this case he will only write the code for the backend but during this time the backend engineer would also have to test his backend code mainly that would be an API which will communicate with the database and send data to the frontend. During this time he will need some way to see whether his backend is sending the correct data to a particular request. In this case he can use an application like Request master to check the responses from his server and also to make it easier for the frontend engineer to create the application as he also has to connect the backend to the frontend and create the user interface according to that.
